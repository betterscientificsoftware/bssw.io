## Lessons from `2027-05-rse-genai.md`

- Multi-author contributor lines can be long. Keep them on one `#### Contributed by ...` line and use linked names separated by commas, with `and` before the final author.
- Date consistency is easy to miss. Compare the filename prefix, publication date, optional `RSS update`, image stem names, and any stated event dates.
- Long perspective articles can use deck text instead of a hero image. Use one or two link-free sentences between `<!-- begin deck -->` and `<!-- end deck -->`.
- Figures should live in `images/` and use HTML image tags. For detailed diagrams, use `class='page lightbox'`, meaningful `alt` text, and a short caption immediately after the tag.
- Acknowledgements, DOI, and cross-posting notes usually belong after the main argument and before author bios.
- Use `## Author bios` for multiple authors. Each bio is a paragraph, optionally starting with a linked author name.
- For formal references, keep human-authored reference definitions above the generated block and let `utils/wikize_refs.py` rewrite the numbered citations. Never patch the generated `sfer-ezikiw` output by hand.
- Hidden metadata should include `Publish`, `Track`, `Pinned`, and `Topics`; `RSS update` is used only when feed behavior needs to be controlled.

## Current Blog Patterns

Minimal deck-text blog:

```markdown
# Title

#### Contributed by [Author](https://github.com/author)

#### Publication date: May 28, 2026

<!-- begin deck -->
One or two sentences.
<!-- end deck -->

Body text.

## Author bio

Bio paragraph.

<!---
Publish: no
Track: Community
Pinned: no
Topics: research software engineers
--->
```

Hero-image blog:

```markdown
# Title

**Hero Image:**

- <img src='../../images/2026-04-everse-hero.png' alt="hero image" />

#### Contributed by [Author](https://github.com/author)

#### Publication date: April 10, 2026
```

Body image:

```markdown
<br>
<img src='../../images/2027-05-rse-genai-figure1.png' class='page lightbox' alt="Figure 1: RSEs bring both software engineering and research expertise."/>Figure 1: RSEs bring both software engineering and research expertise.
<br>
```

Formal citation source block:

```markdown
<!--- References --->

[label-sfer-ezikiw]: https://doi.org/example "Short title {Full bibliographic text.}"
<!-- DO NOT EDIT BELOW HERE. THIS IS ALL AUTO-GENERATED (sfer-ezikiw) -->
```

## Review Checklist

- Title is stable before publication because it drives the final URL slug.
- Filename follows `YYYY-MM-keywords.md` and matches the expected publication month.
- Contributor names and links are complete.
- Publication date uses `Month D, YYYY`.
- Article has deck text or hero image.
- Deck text has no links.
- Body headings start at `##`.
- Images use `../../images/...`, exist in `images/`, and use HTML tags.
- Metadata block has `Publish`, `Track`, and `Topics`.
- Formal citations have been regenerated by `wikize_refs.py`.
- The diff does not include accidental edits to generated references or unrelated files.
